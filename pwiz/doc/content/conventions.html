ProteoWizard project conventions

general

  • All builds are done in the build directory.

gcc-specific

  • gcc makefiles that include src/make_binary.inc will automatically build in build/gcc, with intermediates in build/gcc/project-name.intermediate

msvc-specific

  • MSVC projects use the following directories for builds:
    • output (release): ../../build/msvc-release
    • output (debug): ../../build/msvc-debug
    • intermediate: $(OutDir)/project-name

  • All Windows projects use the CRT in its DLL form. This is done automatically by gcc (mingw). Under MSVC, the CRT library should be specified as multi-threaded [debug] dll.

  • useful preprocessor definitions:
    • _CRT_SECURE_NO_DEPRECATE:   suppress annoying deprecation warnings
    • _SECURE_SCL_THROWS=1:   tell iterator checker to throw rather than crash
    • _USE_MATH_DEFINES:   defines stuff like M_PI in <cmath>
